# Multiplayer Flow

## Purpose

Перевести Game Match из персонального помощника по выбору игр в полноценный сервис для компании друзей.

Сейчас подбор выполняется на основе ответов одного пользователя.

В новой версии подбор должен учитывать предпочтения всех участников комнаты и формировать общий результат.

---

# User Scenario

## Example

Антон хочет поиграть вечером с друзьями.

Он открывает Game Match и создает комнату.

Сервис генерирует ссылку:

```text
/room/A7K92
```

Антон отправляет ссылку друзьям в Discord или Telegram.

Саня, Леха и Рома открывают ссылку, вводят свои никнеймы и проходят опрос.

После того как все завершат опрос, сервис показывает общий результат для всей компании.

---

# Main Flow

## Step 1. Create Room

Пользователь нажимает кнопку:

```text
Создать комнату
```

Система:

* создает комнату
* генерирует уникальный код
* открывает страницу комнаты

Пример:

```text
/room/A7K92
```

---

## Step 2. Share Room

На странице комнаты отображается:

* код комнаты
* ссылка-приглашение
* кнопка копирования ссылки

Пользователь отправляет ссылку друзьям.

---

## Step 3. Join Room

Если пользователь впервые открывает комнату, система показывает экран присоединения.

Заголовок:

```text
Присоединиться к комнате
```

Поле:

```text
Введите никнейм
```

Кнопка:

```text
Продолжить
```

После ввода никнейма участник добавляется в комнату.

---

## Step 4. Room Lobby

После присоединения пользователь попадает в лобби комнаты.

Отображается:

### Информация о комнате

```text
Комната A7K92
```

### Участники

```text
Антон
Саня
Леха
Рома
```

### Статус участников

```text
Антон ✅

Саня ⏳

Леха ⏳

Рома ⏳
```

Статусы:

* ⏳ Опрос не завершен
* ✅ Опрос завершен

---

## Step 5. Quiz

Каждый участник проходит опрос.

После завершения:

* ответы сохраняются
* участник получает статус completed
* пользователь возвращается в комнату

---

## Step 6. Waiting State

Если хотя бы один участник еще не прошел опрос, показывается экран ожидания.

Пример:

```text
Ожидаем остальных игроков

2 из 4 участников завершили опрос
```

Кнопка результатов недоступна.

---

## Step 7. Results Available

Когда все участники завершили опрос:

```text
4 из 4 участников завершили опрос
```

Появляется кнопка:

```text
Показать результаты
```

---

## Step 8. Shared Results

Результат рассчитывается по всем участникам комнаты.

Алгоритм:

1. Собрать ответы всех игроков.
2. Найти совпадающие предпочтения.
3. Найти конфликтующие предпочтения.
4. Рассчитать score для каждой игры.
5. Сформировать рейтинг игр.

Результат отображается один для всей комнаты.

---

# Room States

## Waiting For Players

Игроки присоединяются к комнате.

```text
0+
участников
```

---

## Waiting For Quiz Completion

Все участники присоединились, но не все прошли опрос.

```text
2 из 4 завершили опрос
```

---

## Ready

Все участники завершили опрос.

```text
4 из 4 завершили опрос
```

Появляется возможность открыть результаты.

# Success Criteria

Версия считается успешной если:

* пользователь может создать комнату
* друзья могут присоединиться по ссылке
* каждый участник может пройти опрос
* комната отображает прогресс прохождения
* результат рассчитывается по всем участникам
* несколько пользователей могут использовать сервис одновременно

Главный критерий:

Game Match должен помогать компании друзей договориться во что играть за несколько минут.
