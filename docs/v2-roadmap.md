# Game Match V2 Roadmap

## Goal

Перевести проект из демонстрационного режима в полноценный многопользовательский сервис.

Главное отличие V2:

Сейчас каждый пользователь проходит опрос самостоятельно и получает результат только для себя.

В V2 результат должен формироваться на основе ответов всех участников комнаты.

---

# Current State

Сейчас реализовано:

* Главная страница
* Создание комнаты
* Страница комнаты
* Опрос
* Алгоритм подбора игр
* Страница результатов
* Моковая база игр
* Деплой проекта

Ограничения:

* Нет базы данных
* Нет хранения комнат
* Нет хранения участников
* Нет хранения ответов
* Нет общего результата для компании

---

# V2 Scope

## 1. PostgreSQL

Цель:

Добавить постоянное хранение данных.

Будет храниться:

* комнаты
* участники
* ответы игроков
* результаты подбора

---

## 2. Prisma

Цель:

Создать слой работы с базой данных.

Использовать существующую структуру из database.md.

---

## 3. Real Rooms

Комнаты должны существовать в базе данных.

После создания комнаты:

* генерируется code
* создается запись Room
* комната доступна по ссылке

Пример:

```text
/room/A7K92
```

---

## 4. Room Players

Участники должны присоединяться к комнате.

Новый сценарий:

1. Пользователь открывает ссылку.
2. Вводит никнейм.
3. Создается запись RoomPlayer.
4. Пользователь попадает в комнату.

Страница комнаты должна показывать список участников.

Пример:

```text
Антон ✅

Саня ⏳

Паша ⏳
```

---

## 5. Quiz Persistence

Ответы должны сохраняться в базу данных.

После прохождения опроса:

* создаются записи Answer
* completedQuiz становится true

Ответы больше не должны храниться в sessionStorage.

---

## 6. Shared Room State

Комната должна отображать статус прохождения опроса всеми участниками.

Пример:

```text
3 из 5 игроков завершили опрос
```

---

## 7. Shared Results

Главная функция проекта.

Результаты должны рассчитываться на основе ответов всех участников комнаты.

Алгоритм:

1. Собрать ответы всех игроков.
2. Найти совпадения интересов.
3. Найти конфликты интересов.
4. Рассчитать score для игр.
5. Сформировать общий рейтинг.

Результат отображается один для всей комнаты.

---

# Out Of Scope

Не входят в V2:

* Авторизация через Google
* Профили пользователей
* Tinder-карточки
* История матчей
* Discord интеграция
* Steam синхронизация
* Уведомления
* Система друзей
* Рекомендации на основе истории
* Административная панель

---

# Success Criteria

V2 считается завершенной если:

* Комната хранится в БД
* Игроки могут присоединяться по ссылке
* Ответы сохраняются в БД
* Комната показывает статус участников
* Результаты рассчитываются по всем участникам
* Несколько пользователей могут пользоваться сервисом одновременно
